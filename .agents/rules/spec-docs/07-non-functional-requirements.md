# 7. 非機能要件・その他

## 7.1. スレッドセーフティとタスクスケジューリング（重要）
Minecraftのサーバーアーキテクチャは、ゲームロジック（ワールド操作・エンティティ更新）を単一のメインスレッド（サーバーTickスレッド）で実行する。一方でHTTPサーバーは非同期のワーカースレッドで動作する。
そのため、HTTPリクエストを受け取ったワーカースレッドから直接 `world.setBlockState()` 等のワールド操作を行うと競合が発生し、`ConcurrentModificationException` によるサーバークラッシュやチャンク破損を招く危険がある。
これを防ぐため、HTTPリスナーで受信したリクエストは必ずメインスレッドのタスクキューにエンキュー（スケジュール委譲）し、`CompletableFuture`等を用いてメインスレッドでの処理完了をワーカースレッドが非同期で待機してからレスポンスを返す設計パターンを厳守する。

## 7.2. エラーハンドリング
Modは無効なリクエストに対して適切なHTTPステータスコードとJSONでのエラーメッセージを返し、CLIツールはそれを標準エラー出力としてAIが理解しやすいテキストエラーとして出力する。

## 7.3. セキュリティ
本システムはローカル実行を前提としているが、不正なAPI呼び出しを防ぐため、将来的に簡単な認証ヘッダー（API Key）の追加を想定した設計とする。

## 7.4. クロスプラットフォーム対応と配布
CLIツールはGoで実装され、Goのクロスコンパイル機能を利用して以下のプラットフォーム向けにバイナリを提供する。
- **Windows**: `windows/amd64`, `windows/arm64`
- **Linux**: `linux/amd64`, `linux/arm64`
- **macOS**: `darwin/amd64`, `darwin/arm64` (Apple Silicon)
これにより、ユーザーの環境にGoランタイムやPythonインタープリタをインストールすることなく、単一の実行ファイルのみで動作することを保証する。
