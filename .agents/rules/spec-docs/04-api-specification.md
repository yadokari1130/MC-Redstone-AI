# 4. Minecraftサーバー APIインターフェース仕様

サーバー起動時に内蔵の軽量なHTTPサーバーを立ち上げ、以下のエンドポイントを提供する。

## 4.1. `GET /api/blocks`
指定範囲のブロックリストを取得する。
- **クエリパラメータ**: `x1`, `y1`, `z1`, `x2`, `y2`, `z2` (すべて整数)
- **レスポンス**: 正常時は200 OKとブロックリストのJSON配列を返す。配列要素にはBlockState情報に加え、対応するBlockEntityのNBTデータも含める。空気ブロック (`minecraft:air`) は原則として除外するか、フィルタオプションを設ける。

## 4.2. `POST /api/blocks`
JSON配列として送信されたブロックデータを指定座標へ一括配置する。
- **リクエストボディ**: ブロックデータの配列 (JSON形式)。高速化やバニラ挙動のエミュレートを制御するため、オプションとして「更新フラグ (Flags: 通常時は3, 高速配置時は2など)」の指定をサポートする。
- **レスポンス**: 成功した場合は200 OK。配置不能なブロックや無効な座標が含まれる場合は適宜エラーステータス（400等）を返す。

## 4.3. `POST /api/interact`
特定座標のブロックに対してインタラクト操作を実行する。
- **リクエストボディ**: `{ "x": x座標, "y": y座標, "z": z座標 }`
- **レスポンス**: インタラクト成功時は200 OK。FakePlayerパターンの実行（`onUse`）の戻り値が `SUCCESS` または `CONSUME` であれば成功とする。対象ブロックがインタラクト不能（石ブロックなど）の場合は400 Bad Request。
