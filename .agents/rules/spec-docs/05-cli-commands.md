# 5. CLIコマンド (AI用) 仕様

CLIツールはAIに向けて以下のコマンドを提供する。全コマンド共通で `--url` (デフォルト: `http://localhost:8080`) と `--verbose` (詳細出力) フラグが利用可能。

## 5.1. `get-blocks`
- **概要**: 指定された座標範囲のブロック状況を取得する。
- **引数**:
  - `--pos1`: 開始座標 `"x,y,z"` (整数)
  - `--pos2`: 終了座標 `"x,y,z"` (整数)
  - `--count`: 取得回数 (デフォルト: 1)
  - `--interval`: 取得間隔 (ゲームチック単位)
  - `--include-entities`: 同じ範囲内のエンティティ情報も取得する
- **標準出力**:
  - `--include-entities` 未指定時: `[][]BlockData` (JSON)
  - `--include-entities` 指定時: `[]BlocksAndEntities` (JSON)

## 5.2. `place-blocks`
- **概要**: 考案したレッドストーン回路などの構造物を世界に配置する。
- **引数**:
  - `--blocks`: `PlaceRequest` オブジェクト、またはフェーズ配列 `[PlaceRequest]` のJSON文字列。または `@file.json` 形式のパス。
  - `PlaceRequest` 内の `entities` フィールドでエンティティも同時に配置可能。
- **標準出力**: 配置結果のサマリー。

## 5.3. `kill-entities`
- **概要**: 指定範囲のエンティティを削除する。
- **引数**:
  - `--pos1`: 開始座標 `"x,y,z"` (整数)
  - `--pos2`: 終了座標 `"x,y,z"` (整数)
  - `--type`: (オプション) 削除対象のエンティティタイプ（例: `minecraft:boat`）
- **標準出力**: 削除結果のサマリー（削除件数、スキップ件数）。

## 5.5. `interact-block`
- **概要**: 指定した座標のブロックを操作する。
- **引数**:
  - `--pos`: 対象の座標 `"x,y,z"` (整数)
  - `--pos1`, `--pos2`: (任意) 操作後に状態を確認する範囲
  - `--delay`: (任意) 操作から確認までの待機チック数
- **標準出力**: 操作結果、または指定範囲のブロックデータ。

## 5.6. `fill`
- **概要**: 指定された矩形範囲を特定のブロックで埋める。
- **引数**: `pos1` `pos2` `block_id` (位置引数)
  - `--state`: (任意) ブロック状態のJSON
- **標準出力**: 成功メッセージ。

## 5.7. `drop-items`
- **概要**: アイテムをエンティティとしてドロップする。
- **引数**:
  - `--pos`: ドロップ座標 `"x,y,z"` (小数可)
  - `--items`: アイテム情報の配列JSON、または `@file.json`
- **標準出力**: 成功メッセージ。

## 5.8. `set-inventory`
- **概要**: コンテナブロックのインベントリにアイテムをセットする。
- **引数**:
  - `--pos`: 対象ブロックの座標 `"x,y,z"` (整数)
  - `--items`: アイテム情報の配列JSON、または `@file.json`
- **標準出力**: 成功メッセージ。

## 5.9. `test`
- **概要**: `*_test.yaml` ファイルを実行して回路の動作を検証する。
- **引数**: `<file_or_dir>` (位置引数)
- **標準出力**: テスト結果のサマリー。
