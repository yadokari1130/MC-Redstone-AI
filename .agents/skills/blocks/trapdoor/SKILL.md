# トラップドアの詳細仕様

## Block ID
- `minecraft:oak_trapdoor` (その他、各種木材のトラップドア)
- `minecraft:iron_trapdoor` (鉄のトラップドア)

## プロパティ (Block States)
- `facing`: `north`, `south`, `east`, `west`
  - トラップドアのヒンジ（蝶番）が固定される方角を指定します。
- `half`: `top`, `bottom`
  - トラップドアをブロック内の上半分に設置するか、下半分に設置するかを指定します。
- `open`: `true`, `false`
  - トラップドアが開いているか（垂直状態）、閉じているか（水平状態）を指定します。
- `powered`: `true`, `false`
  - レッドストーン信号を受けているかどうかを示します。
- `waterlogged`: `true`, `false`
  - ブロック内に水が含まれているかどうかを指定します。

## 配置ガイドライン
- `place_blocks` を使用して設置する場合、`facing` と `half` を組み合わせて位置と向きを決定します。
- 例えば、壁に貼り付いた状態で下に開くように設置したい場合は、`facing` を壁の方向に、`half` を `top` に設定します。
- `open` プロパティを `true` にすることで、設置した瞬間から開いた状態にできます。

## 操作 (Interaction)
- **mc_cli_interact_block**: 木材系のトラップドアの `open` 状態を切り替えることができます。
- **鉄のトラップドア**: プレイヤーによる直接操作はできず、レッドストーン信号のみで制御されます。また、鉄のドア同様に **準接続 (QC) の影響は受けません。**
- **オブザーバー検知**: トラップドアの開閉（`open` の変化）は、隣接するオブザーバーで検知可能です。高速なパルスが必要な回路の信号源として利用されることもあります。

## 機能・特性
- 水平状態（閉じている）では、プレイヤーやモブがその上を歩くことができます。
- 垂直状態（開いている）では、通り抜けることができますが、モブはこれを歩けるブロックとして認識するため、落とし穴（トラップ）として利用されます。
- レッドストーン信号を受けると `open` 状態が切り替わり、`powered` が `true` になります。

## 使用方法
- レッドストーン回路において、信号の有無を視覚的に確認するためのインジケーターとして使用されます。
- また、ピストンを使わずにアイテムやモブを落下させる制御機構として頻繁に利用されます。
- `iron_trapdoor` は手動で動かせないため、回路専用のゲートとして適しています。
