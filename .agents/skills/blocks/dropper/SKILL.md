# ドロッパーの詳細仕様

## Block ID
- `minecraft:dropper`

## プロパティ (Block States)
- `facing`: ブロックが向いている方向を指定します (`north`, `south`, `east`, `west`, `up`, `down`)。
  - **【重要】向きの仕様 (Minecraft準拠)**: `facing` は **「アイテムの排出面（口）」** が向いている方角を指します。
  - **具体例**:
    - `facing: "up"` の場合：アイテムは**上方向**へ排出されます。
    - `facing: "south"` の場合：アイテムは**南方向**へ排出されます。
- `triggered`: レッドストーン信号を受けて作動中かどうかを示します (`true`, `false`)。

## 配置ガイドライン
- `place_blocks` を使用する際、方向（`facing`）を指定する場合は基本的に `attaches` を使用してください。これにより、隣接するブロックに対して正しい向きを自動的に設定できます。ただし、空気ブロックに向かって設置する場合など、アタッチ先のブロックが存在しない場合は、通常通り `facing` プロパティで直接方向を指定してください。アイテムはこの方向に排出されます。
- デフォルトではプレイヤーに向かって設置されますが、回路設計時には意図したコンテナや空間を向くように設定してください。

## 操作 (Interaction)
- 右クリック（インタラクト）することで、3x3のインベントリ画面を開くことができます。
- スロットにアイテムを入れることで、作動時にそれらを排出できるようになります。

## 機能・特性
- レッドストーン信号の立ち上がり（ONになった瞬間）で作動し、インベントリ内のアイテムを1つランダムに排出します。
- 排出先にコンテナ（チェストやホッパーなど）がある場合、その中にアイテムを直接移動させます。
- 排出先にコンテナがない場合、アイテムをエンティティとして世界にドロップします。
- ディスペンサーとは異なり、矢を射出したりバケツを使用したりすることはなく、すべて「アイテム」としてドロップします。
- **準接続 (Quasi-connectivity)**: Java版特有の仕様です。詳細は [quasi_connectivity スキル](../../concepts/quasi_connectivity/SKILL.md) を参照してください。
- **インベントリ操作**: **mc_cli_set_inventory** を使用して直接中身をセットできます。
- **コンパレーター出力**: [inventory_detection スキル](../../concepts/inventory_detection/SKILL.md) を参照してください。

## 使用方法
- アイテムの垂直輸送（ドロッパータワー）や、チェストから別のコンテナへの自動移送に使用されます。
- 1回の信号で1つしか排出しないため、連続して排出させるにはレッドストーンクロック回路が必要です。
