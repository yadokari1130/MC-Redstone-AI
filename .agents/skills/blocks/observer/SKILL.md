# 観察者（Observer）の詳細仕様

## Block ID
- `minecraft:observer`

## プロパティ (Block States)
- `facing`: `down`, `up`, `north`, `south`, `west`, `east`
  - **【重要】向きの仕様 (Minecraft準拠)**: `facing` は **「観察面（顔 / 入力元）」** が向いている方角を指します。
  - **出力側**: `facing` で指定した方向の **真反対** へ信号を出力します。
  - **【注意】リピーター等との違い**: リピーターやコンパレーターとは異なり、`facing` は **「入力側」** を指します。
  - **具体例**:
    - `facing: "north"` の場合：**北にあるブロックを観察**し、**南へ出力**します。
    - `facing: "east"` の場合：**東にあるブロックを観察**し、**西へ出力**します。
- `powered`: 変化を検知して信号を出力しているかどうかを示します。
  - 値: `true`, `false`

## 配置ガイドライン
- `place_blocks` (特に `connects`) での設置を推奨。
- **配置時の注意**: `facing` には **「観察したい方向（顔を向ける方向）」** を指定してください。信号は自動的にその反対側へ出力されます。
- **配置例**: 東にあるピストンに信号を送りたい場合は、西にあるブロックを観察する必要があるため、`facing: "west"` と指定します。これにより、出力面（赤い点）は自動的に東を向きます。

## 操作 (Interaction)
- 直接的な右クリック操作（インタラクト）による状態の変化はありません。
- 隣接するブロックの変化を検知することで動作します。

## 機能・特性
- **ブロック更新の検知**: 正面の1ブロックの範囲内でのブロックの状態変化（設置、破壊、状態変化など）を検知します。
- **信号の出力**: 変化を検知すると、背面から強度15のレッドストーン信号を1レッドストーンチック（2ゲームチック）の間出力します。
- **遅延**: 検知から出力までに1レッドストーンチック（2ゲームチック）の遅延があります。

## 使用方法
- **クロック回路**: 2つの観察者を向き合わせることで、高速なパルスを発生させるクロック回路として使用されます。
- **自動化機構**: サトウキビの成長やカボチャの結実を検知して自動で収穫する機構などに利用されます。
- **フライングマシン**: ピストンと組み合わせて、空中や水中を移動する機構の制御に使用されます。
