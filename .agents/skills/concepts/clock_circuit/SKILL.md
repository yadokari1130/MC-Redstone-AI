# クロック回路 (Clock Circuit)

## 概要
クロック回路は、レッドストーン信号を一定の間隔で自動的に出力し続ける回路です。自動収穫機、アイテム仕分け機、タイマー、および複雑な論理回路の同期など、自動化機構の心臓部として機能します。

## 仕様・論理
- **入力**: 通常は不要（自走式）ですが、レバーやブロック動力化による「停止/開始」制御を組み込むことが一般的です。
- **出力**: 一定周期のパルス信号。
- **動作原理**: 信号をループ状に循環させ、その経路中に遅延を設けることで周期を作り出します。

## 主要な構成パターン

### 1. リピータークロック (Repeater Clock)
最も基本的で調整が容易なクロックです。
- **特徴**: リピーターの遅延設定（1〜4 ticks）を組み合わせることで、周期を細かく調整可能。
- **レイアウト例 (2x2)**:
  ```text
  [R1->] [D]    R1: リピーター (右向き)
   [D] [<-R2]   R2: リピーター (左向き)
                D: レッドストーンダスト
  ```
  ※ R1の出力がDに繋がり、そのDがR2の入力、R2の出力が下のD、そのDがR1の入力となるループです。
- **周期**: リピーターの遅延合計の 2 倍。

### 2. エソ式ホッパークロック (Etho Hopper Clock)
長い周期（数秒〜数分）をコンパクトに実現できる非常に信頼性の高い回路です。
- **特徴**: ホッパー内のアイテム数によって周期を正確に設定可能（最大約4.2分）。
- **レイアウト例 (平面図)**:
  ```text
  [RS] [ B ] [ P ] [RSB] [ P ] [ B ] [RS]   B  : 不透過ブロック
  [   ][ C ] [ H ] <---> [ H ] [ C ] [   ]   P  : 粘着ピストン
                                             RSB: レッドストーンブロック
                                             H  : ホッパー (互いに向き合う)
                                             C  : コンパレーター
                                             RS : レッドストーンダスト (Bの上に置く)
  ```
- **構成要素**:
  - `minecraft:hopper` x2
  - `minecraft:redstone_comparator` x2
  - `minecraft:sticky_piston` x2
  - `minecraft:redstone_block` x1
  - 任意の不透過ブロック x2
  - `minecraft:redstone_wire` x2
- **動作原理**: ホッパー間でアイテムが移動しきると、コンパレーターが検知を停止し、ピストンが作動してレッドストーンブロックの位置が切り替わります。

### 3. オブザーバークロック (Observer Clock)
最高速（2ゲームチック間隔）のパルスを発生させる回路です。
- **特徴**: 2つのオブザーバーを向かい合わせるだけで動作。非常に高速（4Hz）。
- **レイアウト**: `[Obs1->][<-Obs2]`

### 4. コンパレータークロック (Comparator Clock)
高速な連射が必要な場合に適した回路です。
- **特徴**: 減算モードを利用。周期は 2ゲームチック (1レッドストーンチック)。
- **レイアウト**: コンパレーターの出力をダストで自身の側面にフィードバック。

## 26.1 における特記事項
- **長期タイマー (Oxidation Timer)**: 酸化段階の変化（数時間単位）をオブザーバーで検知可能です。
- **ティックステップ**: `/tick` コマンドで時間を止めても、レッドストーンロジックのチック数は保持されます。

## 使用ガイドライン
- **負荷 (Lag)**: 高速なクロック（オブザーバー等）は処理負荷（MSPT）を増大させます。
- **停止機構**: レバー等でホッパーをロックしたり、信号を遮断したりして、必要な時だけ稼働するようにしてください。
