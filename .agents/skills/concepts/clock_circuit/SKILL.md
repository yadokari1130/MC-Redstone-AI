# クロック回路 (Clock Circuit)

## 概要
クロック回路は、レッドストーン信号を一定の間隔で自動的に出力し続ける回路です。自動収穫機、アイテム仕分け機、タイマー、および複雑な論理回路の同期など、自動化機構の心臓部として機能します。

## 仕様・論理
- **入力**: 通常は不要（自走式）ですが、レバーやブロック動力化による「停止/開始」制御を組み込むことが一般的です。
- **出力**: 一定周期のパルス信号または持続信号。
- **動作原理**: 信号をループ状に循環させ、その経路中に遅延（リピーターやホッパーの移動時間）を設けることで周期を作り出します。

## 主要な構成パターン

### 1. リピータークロック (Repeater Clock)
最も基本的で調整が容易なクロックです。
- **特徴**: リピーターの遅延設定（1〜4 ticks）を組み合わせることで、周期を細かく調整可能。
- **レイアウト例 (2x2)**:
  ```text
  [R1->] [D]    R1: リピーター (右向き)
   [D] [<-R2]   R2: リピーター (左向き)
                D: レッドストーンダスト
  ```
  ※ R1の出力がDに繋がり、そのDがR2の入力、R2の出力が下のD、そのDがR1の入力となるループです。
- **起動方法**: レッドストーントーチを素早く設置・破壊するか、レバーで一瞬だけ信号を送ります。

### 2. エソ式ホッパークロック (Etho Hopper Clock)
長い周期（数秒〜数分）をコンパクトに実現できる非常に信頼性の高い回路です。
- **特徴**: ホッパー内のアイテム数によって周期を正確に設定可能（最大約4分弱）。
- **レイアウト例 (平面図)**:
  ```text
  [ B ] [ P ] [RSB] [ P ] [ B ]   B  : 不透過ブロック
  [ C ] [ H ] <---> [ H ] [ C ]   P  : 粘着ピストン
                                  RSB: レッドストーンブロック
                                  H  : ホッパー (互いに向き合う)
                                  C  : コンパレーター
  ```
- **構成要素**:
  - `minecraft:hopper` x2
  - `minecraft:redstone_comparator` x2
  - `minecraft:sticky_piston` x2
  - `minecraft:redstone_block` x1
  - 任意の不透過ブロック x2
- **動作原理**: ホッパー間でアイテムが移動しきると、コンパレーターが検知を停止し、ピストンが作動してレッドストーンブロックの位置が切り替わります。

### 3. オブザーバークロック (Observer Clock)
最高速（1-tick間隔）のパルスを発生させる非常にコンパクトな回路です。
- **特徴**: 2つのオブザーバーを向かい合わせるだけで動作します。
- **レイアウト**:
  ```text
  [Obs1->][<-Obs2]   Obs: オブザーバー (顔が互いに向かい合う)
  ```
- **制御**: 片方のオブザーバーをピストンで動かすことで、ON/OFFの切り替えが可能です。

### 4. コンパレータークロック (Comparator Clock)
高速な連射が必要な場合に適した回路です。
- **特徴**: 減算モードを利用し、自身の出力を側面にフィードバックすることで高速点滅させます。
- **レイアウト**:
  ```text
    [D]--[D]      C: コンパレーター (減算モード)
     |    |       D: レッドストーンダスト
  [I]->[C]--[O]   I: 入力信号 (ONの間クロックが動作)
                  O: 出力信号
  ```

### 5. 銅の電球クロック (Copper Bulb Clock)
銅の電球の T-Flip Flop 機能を利用した、状態保持が可能なクロックです。
- **特徴**: 信号が入力されるたびに状態が切り替わるため、バイナリカウンタ（周波数を半分にする）としても機能します。
- **レイアウト**:
  ```text
  [I]->[Bulb][C]->[O]   Bulb: 銅の電球
                        C: コンパレーター (Bulb の状態を読み取る)
  ```

## 26.1 における特記事項
- **長期タイマー (Oxidation Timer)**: 26.1 では銅の酸化段階の変化をオブザーバーで検知可能です。これを利用し、数時間〜数十時間単位の超長期的なランダムタイマー（例：拠点の装飾を自動で切り替える等）を構築できます。
- **クラフター (Crafter)**: 自動作業台のトリガーとして、オブザーバークロックやリピータークロックが頻繁に利用されます。特に 1-tick パルスによる高速動作との相性を考慮する必要があります。

## 使用ガイドライン
- **チャンクロードの影響**: ホッパークロックはチャンクがアンロードされてもアイテム位置が保存されるため、長周期のタイマーとして安定しています。
- **負荷 (Lag)**: 高速なクロック（オブザーバー等）を大量に配置すると、サーバーの処理負荷（MSPT）が増大するため、必要な時だけ稼働するように設計してください。
- **停止機構の設置**: 常に稼働し続ける回路はトラブルの元になるため、必ずレバー等による停止機構（ホッパーのロック、リピーター経路の遮断など）を設けてください。
