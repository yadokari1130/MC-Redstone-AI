# 準接続 (Quasi-connectivity / QC)

## 概要
準接続（QC）は、Minecraft Java Edition固有の仕様であり、特定のブロック（ピストン、ドロッパー、ディスペンサー）が、本来の1x1x1の範囲外である「1つ上の空間」に対する動力供給に反応して動作する現象を指します。この挙動を利用することで、非常にコンパクトな回路や、ブロックの状態変化を検知する「BUD（Block Update Detector）」回路を構築することが可能です。

## 仕様・論理
準接続の影響を受けるブロックは、自身が配置されているマスの**1つ上のマス**が動力を受けている（動力化されている）場合、自身も動力を受けているものとして扱います。

- **対象ブロック**:
  - `minecraft:piston` / `minecraft:sticky_piston`
  - `minecraft:dropper`
  - `minecraft:dispenser`
- **動力源の判定**:
  - 対象ブロックの真上のマスが「強動力」または「弱動力」を受けている。
  - 対象ブロックの斜め上のマスにレッドストーントーチ、リピーター、コンパレーター、レッドストーンブロックなどの動力源が隣接している（真上のマスを動力化する条件と同じ）。

### 動作原理
ゲーム内の判定ロジックにおいて、これらのブロックは「2マスの高さを持つドア」と同様の動力判定アルゴリズムを流用しているため、上部の空間への動力を自身の動力として認識します。

### アップデートの必要性（BUD挙動）
準接続によって動力を受けている場合、動力源が対象ブロックに直接隣接していないことが多いため、動力源がON/OFFになっても対象ブロックに「ブロックアップデート（Block Update）」が届かないことがあります。この場合、対象ブロックは「動力状態が変わったが、まだ動かない」という待機状態になります。隣接するブロックが設置・破壊されたり、状態が変わったりした瞬間に、対象ブロックは自身の状態を再確認し、動作を開始します。

## 推奨レイアウト

### 1. 基本的な準接続（斜め上からの動力）
以下の配置では、レッドストーンブロックがピストンの斜め上にありますが、ピストンは伸長します。

```text
[R] : レッドストーンブロック
[P] : ピストン（横向き）
[ ] : 空白

[R][ ]
[ ][P]
```

### 2. 垂直方向の準接続（1マス空けた動力）
真上のマスのさらに上に動力源がある場合も反応します。

```text
[B] : 不透過ブロック
[L] : レバー（ON）
[P] : ピストン（上向き）

 [L]
 [B]
 [ ] (1マスの隙間)
 [P]
```
※この場合、レバーをONにしただけではピストンは動きませんが、[P]の隣にブロックを置くなどのアップデートを与えると伸長します。

## 26.1 における特記事項
- **仕様の継続**: 26.1 においても、準接続は Java 版の重要な意図された仕様として維持されています。
- **対象外のブロック**: 1.21 で追加された **クラフター (Crafter)** や **銅の電球 (Copper Bulb)** は、準接続の影響を受けません。これらは自身のマスに直接動力を受ける必要があります。
- **銅の電球 (Copper Bulb) の活用**: 銅の電球は準接続の影響を受けませんが、状態が変化する際に強力なブロックアップデートを周囲に発します。これを利用して、準接続で待機状態にあるピストンを意図的に作動させる「アップデート用ブロック」として利用可能です。

## 使用ガイドライン
- **コンパクト化**: 垂直方向に信号を伝える際、リピーターなどを使わずにピストンを重ねて配置する場合などに必須の知識です。
- **意図しない動作の回避**: ピストンの上にレッドストーンダストを敷く際、そのダストがピストンの「斜め上の空間」を動力化してしまうため、隣のピストンが意図せず動いてしまうことがあります。これを防ぐには、不透過ブロックの代わりに下半分のハーフブロックなどを使用する工夫が必要です。
- **BUDスイッチ**: ブロックの設置や作動を検知して信号を出す機構を作る際、準接続による「待機状態」をあえて作ることで、センサーとして機能させることができます。
