# 準接続 (Quasi-connectivity / QC)

## 概要
準接続（QC）は、Minecraft Java Edition固有の仕様であり、特定のブロック（ピストン、ドロッパー、ディスペンサー）が、本来の1x1x1の範囲外である「1つ上の空間」に対する動力供給に反応して動作する現象を指します。この挙動を利用することで、非常にコンパクトな回路や、ブロックの状態変化を検知する「BUD（Block Update Detector）」回路を構築することが可能です。

## 仕様・論理
準接続の影響を受けるブロックは、自身が配置されているマスの**1つ上のマス**が動力を受けている（動力化されている）場合、自身も動力を受けているものとして扱います。

- **対象ブロック**:
  - `minecraft:piston` / `minecraft:sticky_piston`
  - `minecraft:dropper`
  - `minecraft:dispenser`
- **対象外のブロック (重要)**:
  - `minecraft:iron_door`, `minecraft:iron_trapdoor` (ドアロジックを持ちますが、QC はピストン系特有の挙動です)
  - `minecraft:fence_gate`
  - `minecraft:redstone_lamp`
  - `minecraft:crafter`, `minecraft:copper_bulb`

### 動作原理
ゲーム内の判定ロジックにおいて、これらのブロックは「2マスの高さを持つドア」と同様の動力判定アルゴリズムを流用しているため、上部の空間への動力を自身の動力として認識します。ただし、ピストン等はドアと異なり、上部が動力化されても「自分自身への更新」が届かない場合に状態が変わらないため、BUD として利用できます。

### アップデートの必要性（BUD挙動）
準接続によって動力を受けている場合、動力源が対象ブロックに直接隣接していないことが多いため、動力源がON/OFFになっても対象ブロックに「ブロックアップデート（Block Update）」が届かないことがあります。

## 26.1 における特記事項
- **対象外のブロック**: 1.21 で追加された **クラフター (Crafter)** や **銅の電球 (Copper Bulb)** は、準接続の影響を受けません。
- **オブザーバー (Observer)**: オブザーバーも準接続の影響を受けず、自身の「顔」の正面 1 ブロックのみを監視します。
- **銅の電球による更新**: 銅の電球は、点灯/消灯の瞬間に周囲 6 面にブロックアップデートを発生させます。これを利用して、準接続で待機しているピストンを強制的に動かすことができます。

## 使用ガイドライン
- **コンパクト化**: 垂直方向に信号を伝える際、リピーターなどを使わずにピストンを重ねて配置する場合などに必須の知識です。
- **意図しない動作の回避**: ピストンの上にレッドストーンダストを敷く際、そのダストがピストンの「斜め上の空間」を動力化してしまうため、隣のピストンが意図せず動いてしまうことがあります。これを防ぐには、不透過ブロックの代わりに下半分のハーフブロックなどを使用する工夫が必要です。
- **BUDスイッチ**: ブロックの設置や作動を検知して信号を出す機構を作る際、準接続による「待機状態」をあえて作ることで、センサーとして機能させることができます。
