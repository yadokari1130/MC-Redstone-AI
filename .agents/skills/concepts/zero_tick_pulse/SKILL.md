# 0ティックパルス (0-Tick Pulse)

## 概要
0ティックパルスとは、単一のゲームティック（1/20秒）内で信号がONになり、かつOFFになる極めて短いレッドストーン信号のことです。
Minecraftの内部処理では、同じティック内でもブロックの更新には優先順位があり、順番に処理されます。この処理順序の隙間を利用することで、論理的には持続時間「0」のパルスを生成できます。

主な用途は、**粘着ピストンの「引き戻し処理」をキャンセルさせ、ブロックをその場に置いてこさせる（パルスエッジでのブロック吐き出し）**ことです。

## 仕様・論理
- **入力**: 1ティック以上のレッドストーン信号、または特定のブロック更新。
- **出力**: 0ティックのパルス（見た目には一瞬だけ信号が走る）。
- **動作原理**:
  - ゲームティック内の更新順序（Tile TickやBlock Updateの優先度）を利用します。
  - ピストンが信号を受け取って「伸びる」処理が開始された直後、同じティック内で信号がOFFになると、ピストンは「伸びきった」判定になりますが、「引き戻す」ための信号が既に消えているため、ヘッドだけが戻りブロックが残されます。

### ピストンの挙動
- **粘着ピストン**: 0ティックパルスを受けると、ブロックを瞬時に押し出し、引き戻さずにその場に残します（もう一度パルスを送ると回収します）。
- **通常のピストン**: 瞬時に押し出し、即座に縮みます。

## 推奨レイアウト

### 1. 更新優先度型（高信頼構成）
Java版の「リピーター」と「コンパレーター」の内部処理優先度の差を利用した、座標や方角に依存しない信頼性の高い構成です。

```text
           (A) [リピーター (1遅延)] ----> [0ティック出力]
 [入力] ----+
           (B) [コンパレーター] --- [粘着ピストン]
                                           |
                                     [ブロックを動かしてAの入力を遮断]
```
- **動作原理**:
  1. Java版の内部処理優先度において、リピーターはコンパレーターよりも先に更新（Tile Tick）を処理します。
  2. 入力信号が入ると、まずリピーター(A)が ON になり出力を開始します。
  3. 同じティック内の直後にコンパレーター(B)がピストンを起動し、リピーターへの入力（または出力）を物理的に遮断します。
  4. 結果として、リピーターの出力は 1ティックに満たない極めて短い時間（0ティック）のみ維持されます。

### 2. 簡易型（ボタン直付け）
最も単純な確認方法です。
- **構成**: 粘着ピストンの「側面」に直接ボタンを設置し、そのピストンがブロックを保持するようにします。
- **動作原理**: ボタンを押すとピストンが伸び始めますが、ピストンの「伸長中」の状態では側面にあるボタンが支えを失いアイテム化して外れます。これにより動力が瞬時に切断され、0ティックパルスが発生します。

## 26.1 における特記事項
- **仕様の維持**: Minecraft 26.1 においても、0ティックパルスの挙動（粘着ピストンのブロック吐き出し等）は Java 版の標準仕様として維持されています。
- **レッドストーンの実験 (Redstone Experiments)**: 実験的機能が有効なワールドでは、信号の伝達ロジックが変更されるため、従来の 0ティック生成回路が動作しなくなる、あるいは挙動が変わる可能性があります。標準的なサバイバルやマルチサーバーでは、従来の挙動がデフォルトです。
- **銅の電球 (Copper Bulb)**: 銅の電球は 1レッドストーンチックの遅延を持つため、0ティックパルスの「入力」として使うことは可能ですが、電球の「出力」を 0ティックにするには別途短縮回路が必要です。

## 使用ガイドライン
- **高速ピストン輸送**: ブロックを 1ティックごとに 1マス移動させるような、極めて高速な輸送機構に使用します。
- **パルス短縮**: 長い信号を瞬時のパルスに変換し、回路の応答速度を極限まで高める際に有効です。
- **注意点**: 0ティックパルスは Java 版特有の挙動であり、統合版（Bedrock Edition）では動作しません。また、座標依存（Locationality）によって動作が不安定になる場合があるため、信頼性の高い設計を選択することが重要です。
- **信号の引き伸ばし**: 0ティックパルスをリピーターやコンパレーターに入力すると、信号が 1〜2ティックの通常のパルスに引き延ばされて出力されます。
