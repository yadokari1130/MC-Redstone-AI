# BUD (Block Update Detector) - ブロック更新検知器

## 概要
BUD (Block Update Detector) は、隣接するブロックの状態変化（設置、破壊、レッドストーン信号の変化など）を検知してレッドストーン信号を出力する回路です。Java版特有の仕様である「準接続 (Quasi-Connectivity)」を利用することで、オブザーバー（観察者）を使用せずにブロック更新を検知することができます。

## 仕様・論理
BUD は主に Java 版の仕様である「準接続 (Quasi-Connectivity)」を利用します。
- **準接続の詳細**: 詳細は [quasi_connectivity スキル](../quasi_connectivity/SKILL.md) を参照してください。
- **入力**: センサー部（通常はピストン）に隣接するブロックの更新。
- **動作原理**: 
    1. ピストン等のブロックが、その 1 マス上の空間が動力化されている状態（準接続）に置かれます。
    2. ブロックは「動力源があるはず」なのに、自分自身が更新されるまでそのことに気づかず、現在の状態を維持します。
    3. 隣接ブロックが更新されると、状態を再確認し、動作（伸びる等）します。

### オブザーバー (Observer) との使い分け
現代の Minecraft では、[block_update_detection](../block_update_detection/SKILL.md) で解説されている「オブザーバー」が公式な BUD として機能します。

| 特徴 | オブザーバー (Observer) | 伝統的 BUD (準接続式) |
| :--- | :--- | :--- |
| **設置** | 1ブロックで完結 | 数ブロックの回路が必要 |
| **検知方向** | 顔の正面のみ | 周囲 6 面 + 準接続範囲 |
| **出力** | 1ティックのパルス | 自由（トグル、持続など） |
| **主な用途** | 標準的な検知、クロック | 隠しスイッチ、高度なピストン制御 |

## 推奨レイアウト

### 1. 基本的なピストン型BUD（トグル式）
ピストンが準接続によって「本来なら伸びるべき状態」で待機し、更新を受けると伸びるレイアウトです。

```
[側面図]
   R (レッドストーンダスト/動力源)
   B (不透過ブロック)
   . (空き)
   P (粘着ピストン - 横向き)
   S (検知対象ブロック)

※ RがBを動力化しており、Pはその1マス上（Bの隣）の空間から
   準接続を受けているため、更新があるとPが伸びます。
```

### 2. 自己リセット型BUD（パルス出力）
検知後に自身で回路を遮断し、元の状態に戻ることでパルス信号を出力するタイプです。

```
[側面図]
   R (レッドストーンブロック)
   B (不透過ブロック)
   P (粘着ピストン - 上向き)
   A (空気)

1. Pの上にBがあり、その上にRがある状態。
2. RはPを準接続で動力化していますが、Pは更新されるまで動きません。
3. Pの隣でブロック更新があると、Pが伸びてBを押し上げます。
4. Bが移動することでRによる動力供給が止まり（またはピストン自身が更新され）、Pが縮んで元の状態に戻ります。
```

## 必要な主要ブロック
- `minecraft:sticky_piston` (粘着ピストン)
- `minecraft:redstone_torch` または `minecraft:redstone_block` (動力源)
- `minecraft:redstone_dust`
- 不透過ブロック (石、滑らかな石など)

## 26.1 における特記事項
- **準接続の維持**: 26.1 においても準接続（Quasi-Connectivity）は Java Edition の仕様として維持されており、BUD 回路は引き続き動作します。
- **オブザーバーとの使い分け**: 現在は「オブザーバー」が公式な BUD として機能しますが、準接続を利用した BUD は、オブザーバーが設置できない場所や、特定の方向・距離からの更新のみを検知したい場合に依然として有用です。
- **銅の電球 (Copper Bulb)**: BUD の出力を銅の電球に繋ぐことで、非常にコンパクトな T-フリップフロップ（状態保持）付きの検知システムを構築できます。

## 使用ガイドライン
- **設置方向**: 準接続は「上方向」の動力を参照するため、動力源は必ず検知対象のブロックの斜め上や 2マス上に配置する必要があります。
- **用途**:
    - 自動サトウキビ収穫機（成長を検知）。
    - 隠し扉のスイッチ（壁の裏での設置・破壊を検知）。
    - 複雑なレッドストーン回路内での特定ブロックの状態監視。
- **注意点**: 周囲の意図しないレッドストーン信号の変化も「更新」として検知してしまうため、他の回路から適切に隔離する必要があります。
