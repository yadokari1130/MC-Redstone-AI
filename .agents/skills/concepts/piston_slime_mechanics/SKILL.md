# ピストン・スライム機構 (Piston & Slime Mechanisms)

## 概要
ピストンと、周囲のブロックを接着して移動させる「スライムブロック」および「ハチミツブロック」を組み合わせることで、自動で移動する飛行マシンや大型の隠し扉などを構築する技術です。

## 仕様・論理

### 1. ピストンの押し出し制限 (Push Limit)
- **制限数**: 1つのピストン（通常のピストンおよび粘着ピストン）が一度に動かせるブロックの最大数は **12ブロック** です。
- **計算対象**: ピストンの正面にある直接のブロックだけでなく、スライムブロック等によって接着され、連鎖的に動くすべてのブロックが含まれます。
- **動作失敗**: 13ブロック以上を動かそうとした場合、ピストンは伸長（または収縮）しません。

### 2. 粘着特性と非干渉性
- **スライムブロック (Slime Block)**: 隣接するほぼすべてのブロックを接着し、一緒に移動させます。
- **ハチミツブロック (Honey Block)**: スライムブロックと同様に隣接ブロックを接着しますが、**スライムブロックとは互いに接着しない** という重要な特性を持ちます。
- **活用のメリット**: スライムブロックの構造とハチミツブロックの構造を隣接させることで、互いに干渉（接着）することなく、より複雑でコンパクトな並列機構を構築できます。

### 3. ブロックの可動性分類 (Java Edition)
| カテゴリ | ブロック例 | 挙動 |
| :--- | :--- | :--- |
| **可動 (Moveable)** | 石、土、ガラス、**銅の電球**、コンクリート等 | 押し引き可能、接着される。 |
| **非粘着・可動 (Non-sticky)** | **釉薬テラコッタ** | 押し引きは可能だが、スライム等の接着効果を受けない。 |
| **不動 (Immovable)** | 黒曜石、岩盤、**クラフター**、**試練スポナー**、**宝物庫**、チェスト等のブロックエンティティ | ピストンで動かすことができず、接着もされない。 |

## 推奨レイアウト：基本の飛行マシン（エンジン部分）
観察者（オブザーバー）と粘着ピストンを組み合わせた、一方向に無限に移動する基本的なエンジンです。

```text
    [O] [P] [S] [S]
    [S] [S] [p] [O]

 [O]: 観察者 (顔が外側、出力が内側)
 [P]: 粘着ピストン (進行方向を向く)
 [p]: 粘着ピストン (後ろを向く)
 [S]: スライムブロック (またはハチミツブロック)
```

1. 前方のオブザーバーが更新を検知し、粘着ピストン[P]を動かして後方の[S]グループを押し出します。
2. 後方のオブザーバーが移動による更新を検知し、粘着ピストン[p]を動かして前方の[S]グループを引き寄せます。
3. これが繰り返され、連続的な移動が発生します。

## 26.1 における特記事項
- **銅の電球 (Copper Bulb)**: 
  - **可動**です。ブロックエンティティではないため、ピストンで自由に動かせます。
  - 状態（ON/OFF）を保持したまま移動できるため、飛行マシン上のフラグ管理に最適です。
- **クラフター (Crafter) / 試練スポナー (Trial Spawner) / 宝物庫 (Vault)**: 
  - すべて **不動** です。飛行マシンに載せて移動させることはできません。
- **粘着ピストンの1ティックパルス (1-tick Pulse)**:
  - Java版固有の挙動（ブロックの置き去り）は 26.1 の標準設定でも維持されています。ただし、ワールド設定で「レッドストーンの実験」を有効にしている場合は挙動が変わる可能性があるため注意が必要です。

## 使用ガイドライン
- **押し出し制限の管理**: 複雑な機構を作る際は、常に「このピストンが動かすブロックの合計」を数え、12個を超えないように設計してください。
- **黒曜石の活用**: 機構を停止させたい場所（ドック）に黒曜石などの不動ブロックを配置することで、強制的にピストンの動作を止めることができます。
- **エンティティへの影響**: ハチミツブロックは上に乗っているMobやプレイヤーを一緒に運びますが、スライムブロックは上に乗っているものを跳ね飛ばす特性があります。用途に応じて使い分けてください。
