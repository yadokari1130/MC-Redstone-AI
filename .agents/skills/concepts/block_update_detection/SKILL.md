# ブロック更新検知 (Block Update Detection - Observer)

## 概要
ブロック更新検知（BUD）は、特定のブロックの状態変化（設置、破壊、成長、状態遷移など）をトリガーにしてレッドストーン信号を発生させる技術です。Java Edition においては、**オブザーバー (Observer)** がこの役割を担う標準的なコンポーネントです。

オブザーバーは「顔」の面で接しているブロックの変更を監視し、変化を検知すると「背中」から短い信号を出力します。

## 仕様・論理
- **入力**: 監視対象ブロックの状態変化（Block State Change）。物理的な破壊/設置のほか、以下の状態変化も検知します。
    - `powered`: レッドストーン入力の有無（リピーター等）
    - `lit`: ランプや火の点灯
    - `waterlogged`: 水没状態の変化
    - `age`, `stage`: 農作物や苗木の成長
- **出力**: 背面の出力端子から**信号強度15（強動力）**のパルスを出力します。
- **ブロック状態**: `facing` プロパティで**出力面（背中）**の向きを指定します。顔はその反対側を向きます。
- **動作原理**:
    - 検知から出力までに **1レッドストーンチック (2ゲームチック)** の遅延があります。
    - 出力されるパルスの長さは **1レッドストーンチック (2ゲームチック)** です。

## 推奨レイアウト

### 基本配置
```text
[監視対象] <- [オブザーバー(顔)] [オブザーバー(背)] -> [出力先]
```
- **配置時の注意**: オブザーバーを設置する際、ターゲットとするブロックの「反対側」から設置すると、顔がターゲットを向きます。

### オブザーバー・クロック (Observer Clock)
2基のオブザーバーを向かい合わせに配置することで、相互に状態変化を検知し続け、高速なパルス（2ゲームチック間隔）を生成します。
```text
  [オブザーバー A] -> <- [オブザーバー B]
  (背)        (顔)    (顔)        (背)
```

### 必要な主要ブロック
- `minecraft:observer`: 検知の核となるブロック。
- `minecraft:sticky_piston`: オブザーバーを動かして検知のON/OFFを切り替える際に使用。

## 26.1 における特記事項
- **バージョン表記**: `YY.D.H` 形式（現在は `26.1`）に基づき、最新のブロック挙動を反映しています。
- **銅の酸化検知**: 銅ブロック等の酸化段階の変化（無酸化→錆びかけ等）はブロック状態の更新として扱われるため、オブザーバーで検知可能です。
- **音符ブロックのトランペット (Trumpet)**: 銅ブロックの上に音符ブロックを置くとトランペットの音が鳴ります。この際、下の銅ブロックの酸化度によって音色が変化しますが、酸化による状態変化自体をオブザーバーで検知し、自動演奏システム等に組み込むことが可能です。
- **銅の電球 (Copper Bulb)**: 状態が変化する（ON/OFF）際にブロック更新を発生させるため、オブザーバーと組み合わせることで Tフリップフロップをコンパクトに構成できます。
- **クラフター (Crafter)**: アイテム搬入・搬出、およびクラフト実行時の状態変化をオブザーバーで検知可能です。

## 使用ガイドライン
- **パルスの短さへの注意**: 1レッドストーンチックのパルスは非常に短いため、粘着ピストンの「ブロックの置き去り（1-tick pulse behavior）」を意図的に引き起こすことができます。
- **意図しないループ**: オブザーバーの出力が自身の監視対象ブロックを変化させてしまうと、意図しない無限ループ（クロック）が発生するため注意が必要です。
- **移動による検知**: オブザーバー自身がピストンで動かされた際、移動完了後に周囲の状況を再検知し、パルスを発信します。
