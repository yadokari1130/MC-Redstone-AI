# ブロック更新順序 (Block Update Order)

## 概要
ブロック更新順序とは、Minecraft内で複数のイベントが同じゲームティック（Tick）内に発生した際、それらがどの順番で処理されるかを決定する仕組みです。特にレッドストーン回路において、同時に動くはずのピストンやレッドストーンダストが予期せぬ動作をする原因（座標依存や方角依存）を理解し、制御するために不可欠な知識です。

## 仕様・論理
Minecraft Java Edition 1.21.2 以降では、レッドストーンワイヤー（ダスト）の更新ロジックが大幅に改善され、より予測可能になっています。

### 1. 標準的な近傍更新（Neighbor Update / NC）
ブロックの状態が変化したとき、周囲の6ブロックに送られる更新（Neighbor Changed）の標準的な順序は以下の通りです。
1.  **西 (West / -X)**
2.  **東 (East / +X)**
3.  **下 (Down / -Y)**
4.  **上 (Up / +Y)**
5.  **北 (North / -Z)**
6.  **南 (South / +Z)**
これを「**W-E-D-U-N-S**」順と呼びます。レバーやトーチなどの動力源はこの順序で周囲を更新するため、回路の向きによって動作が変わる「方角依存（Directionality）」の原因となります。

### 2. レッドストーンワイヤーの更新（1.21.2+ 以降）
新しいワイヤー更新アルゴリズムでは、以下の優先順位で更新が行われます。
-   **距離優先 (Distance Priority)**: 動力源に近いワイヤーから順に更新されます。
-   **タイブレーク (Left-First)**: 信号の進行方向に対して「左側」にあるワイヤーが優先されます。
-   **ランダム化**: 進行方向が特定できない場合（真上/真下からの動力など）は、ランダムに選択されます。
-   **周囲への更新順序**: ワイヤー自体が周囲を更新する際は、そのワイヤーに「信号が到達した方向」を基準に以下の順で更新されます。
    1.  **後ろ (Back)**
    2.  **前 (Front)**
    3.  **左 (Left)**
    4.  **右 (Right)**
    5.  **下 (Down)**
    6.  **上 (Up)**

### 3. 座標依存 (Locationality)
同じティック内に発生する「ブロックイベント」（ピストンの伸縮など）は、世界座標に基づいて処理順序が決まることがあります。
-   通常、座標の小さい順（-X, -Y, -Z 優先）で処理されることが多いですが、ハッシュ値に基づいたランダムに近い順序になる場合もあります。
-   これにより、同じ構成の回路でも設置場所（座標）によって挙動が変わる「座標依存」が発生します。

## 推奨レイアウトと制御手法

### 決定論的な順序の確保
更新順序の問題を避けるには、以下の手法が推奨されます。

-   **遅延（Delay）の導入**: リピーターを使用して1ティック以上の差をつけることで、更新順序の競合を物理的に回避します。
-   **単一の動力伝達パス**: 信号が分岐して合流するような構成を避け、常に一方から他方へ流れるように設計します。
-   **1.21.2+ の新仕様を活用**: ワイヤーの「左優先」や「距離優先」を利用して、ピストンの動作順を制御できます。

### 必要な主要ブロック
- `minecraft:redstone_wire`
- `minecraft:repeater` (遅延による順序制御)
- `minecraft:observer` (更新の検知と伝達)

## 26.1 における特記事項
- **標準挙動と実験的機能**: 26.1 においても、新しいレッドストーン更新ロジックは「実験的機能（Redstone Experiments）」として提供されています。これを有効にしない限り、従来の W-E-D-U-N-S 順と座標依存が適用されます。
- **予測可能性の向上**: 実験的機能を有効にすることで、ワイヤーの更新が「距離優先」になり、座標依存の多くが解消されます。技術的な回路設計においては、この機能を有効にすることが推奨されます。
- **パフォーマンス**: 26.1 では内部的なレンダリングやデータ構造の最適化が行われていますが、レッドストーンの論理計算自体は実験的機能の有無に依存します。

## 使用ガイドライン
- **同時動作の回避**: 2つのピストンが同じブロックを奪い合うような設計は避け、必ずリピーターで順序を明確にします。
- **向きの確認**: 標準設定では NC は「W-E-D-U-N-S」順であるため、非ワイヤー系動力源（トーチ、レバー、オブザーバー等）を直接隣接させる場合は方角に注意してください。
- **「実験的機能」の活用**: 回路を座標に依存させたくない場合は、ワールド設定でレッドストーンの実験を有効にしてください。
